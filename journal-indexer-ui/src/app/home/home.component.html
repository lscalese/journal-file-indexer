• This is a tools to index your journal file into the database.
  • The goal is :
    • Search data
    • Restore data in new location (data are not restored in the original global)
    • Show stats
  • Indexed journal data are considered as "temporary data".
  • So, indexed data are stored in IRISTEMP database.
This tools should not used directly on a live system.
The good practice is to index journal file on a test system.
Data are stored in IRISTEMP, make sure you have enough space before indexing a file.
1 GB journal file can take up to 5 Gb in database.
By Default Journal Indexer keep 5 indexed journal max, if a 6th is indexed, the oldest is automatically deleted.
